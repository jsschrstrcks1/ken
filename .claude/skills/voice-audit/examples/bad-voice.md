# Bad Voice — example passage

Same hypothetical handoff, written in the failure mode `voice-audit` is designed to catch.

---

## Where We Are

We've been making incredible progress on the Manatee Creek migration — it's been an exciting journey, and the spreadsheet is really starting to come together. The team has done amazing work pushing through the technical challenges.

We're roughly two-thirds of the way home. There are still a few outstanding items, but the trajectory is genuinely positive.

## What's Left

A handful of tabs remain in flight. They're blocked on what is essentially a schema alignment question, but we believe with focused effort we can resolve these in the next session.

## Vibes

Strong. The fundamentals are sound. We just need to keep the momentum going and trust the process.

---

Why this fails voice-audit:

- Zero file paths, command names, sheet IDs, or line numbers.
- Mantra phrases ("incredible progress," "exciting journey," "trust the process," "the fundamentals are sound").
- Hedge stacks ("roughly two-thirds," "we believe," "essentially," "a handful").
- Adversarial-or-cheerleading register where utility prose is needed.
- A new session reading this file cannot tell which tabs are blocked, what the schema disagreement is, or what command to run next. The handoff has lost its job.
- "Vibes" as a section heading is the diagnostic giveaway: when a documentation file has a section that exists only to feel a certain way, voice-audit fires.
