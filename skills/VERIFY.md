# Verification Before Completion

*Adapted from jsschrstrcks1/ken `.claude/skills/verification-before-completion/SKILL.md` for OpenClaw/Skynet.*

---

## The Iron Law

```
NO COMPLETION CLAIMS WITHOUT FRESH VERIFICATION EVIDENCE
```

Claiming work is complete without verification is dishonesty, not efficiency.
Evidence before claims, always.

---

## The Gate

Before claiming any task is done:

1. **IDENTIFY** — What proves this claim? Which file, command, or check?
2. **RUN** — Execute the full check (fresh, not cached)
3. **READ** — Read the actual output
4. **VERIFY** — Does output confirm the claim?
   - If NO: State actual status with evidence
   - If YES: State claim WITH evidence
5. **ONLY THEN** — Make the claim

Skip any step = claiming without evidence.

---

## Common Claims and What They Require

| Claim | What It Requires | Not Sufficient |
|-------|-----------------|----------------|
| "File updated" | Show the actual content | "I wrote it" |
| "Tests pass" | Test output: 0 failures | "Should pass" |
| "Bug fixed" | Reproduce → confirm fixed | Code changed |
| "Done" | Line-by-line against the original request | Task feels complete |
| "Agent completed" | Check the actual output | Agent reports "success" |

---

## Red Flags — Stop Before Claiming

- Using "should", "probably", "seems to"
- "I think this is correct"
- About to claim done without showing the output
- Trusting a tool's self-report without checking
- "Just this once"
- **Any wording implying success without having verified**
